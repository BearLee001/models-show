# 图像修复框架前端界面

> 本项目完全是为了展示 Codeformer ref-ldm 等图像修复框架效果存在，界面比较粗糙，基本都是 AI 生成的，主要使用 Gradio 构建
>
> 非模型学习相关的须知内容都在本 README 中（比如前后端架构、拓展方法...)。另外三个工程对应的 repo 实现**相应的服务器和可能要做的微调**即可

## 整体架构

### 做了什么？

在每个模型的项目里引入了 **FastApi** 依赖，借助该库快速搭建出封装了模型能力的 python 服务器，可以接受请求，并根据请求使用模型能力

### 为什么要这么做？

1. 图像修复框架都是 python 工程，尽可能使用 python 搭建服务器，**避免跨语言调用的理解和学习成本**
2. 希望有一个前端能够**展示多个模型**的能力，但是每个模型的 python 环境不一样，需要中间层来适配
3. 架构设计弱耦合带来了极好的拓展性和调试环境（只需要关注服务之间传递的信息）
